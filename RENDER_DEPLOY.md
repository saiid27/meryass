# Render deploy

## What this repo provides

`render.yaml` creates:

- `meryass-backend`: Python web service running `backend/app.py`
- `meryass-db`: PostgreSQL database wired into `DATABASE_URL`

The backend exposes `/health` for Render health checks.

## Deploy steps

1. Push this repo to GitHub.
2. In Render, create a new Blueprint from the repo.
3. Let Render read `render.yaml`.
4. Deploy.
5. Copy the backend URL, for example:

```text
https://meryass-backend.onrender.com
```

6. Run Flutter with that API host:

```bash
cd frontend
flutter run --dart-define=API_HOST=https://meryass-backend.onrender.com
```

## Production notes

- `SECRET_KEY` and `JWT_SECRET_KEY` are generated by Render.
- `FLASK_ENV=production` disables development bots.
- `CORS_ORIGINS` is currently `*` for easy mobile/web testing. Restrict it later if a fixed web domain is used.
- Free services may sleep when idle; first request can be slow.
